Abstract

Data ecosystems are complex networks of autonomous participants that exchange data, services and other resources while preserving sovereignty and interoperability. To design, compare and operate such ecosystems it is essential to model both their dynamic behaviour (the processes that drive data sharing) and their static architecture (the technical components that realise those processes) in a technology-agnostic way.

The Data Ecosystem Description Language (DEDL) is an information-model-based abstract language that captures these two dimensions; its core concepts are Process (e.g., Transacting, Onboarding, Offboarding) and TechnicalDataEcosystemElement (e.g., Identity and Access Management, Storage, Communication, Interface, Run Time Environment). By combining these building blocks, DEDL can describe centralized, federated, or hybrid data ecosystem realizations in a uniform, extensible format.
